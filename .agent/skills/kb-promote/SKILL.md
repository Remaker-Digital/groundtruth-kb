# Promote Specification Status

Safely advance a specification through its lifecycle with assertion-gated validation.

**Arguments:** `$ARGUMENTS[0]` = spec ID (e.g., `SPEC-1803`), `$ARGUMENTS[1]` = target status (optional, defaults to next in lifecycle).

## Step 1: Read Current State

```python
import sys; sys.path.insert(0, 'tools/knowledge-db')
from db import KnowledgeDB
db = KnowledgeDB()

spec = db.get_spec("$0")
print(f"ID: {spec['id']}  Status: {spec['status']}  Version: {spec['version']}")
assertions = spec.get('assertions') or []
print(f"Assertions: {len(assertions)}")
```

## Step 2: Determine Target Status

If no target provided: `specified -> implemented -> verified`. `verified` and `retired` are terminal.

## Step 3: Validate Promotion

See `references/validation-rules.md` for the full validation matrix by target status.

**Key gates:**
- **specified -> implemented:** Assertions pass + no blocking WIs + code exists + **at least 1 linked test (GOV-12)**
- **implemented -> verified:** All above + tests exist (GOV-12) + tests pass + live verification (GOV-10)
- **any -> retired:** Owner approval required (GOV-02)

**GOV-12 enforcement (WI-1477):** Before any promotion, check:
```python
tests = db.get_tests_for_spec("$0")
non_stale = [t for t in tests if t.get('last_result') != 'stale']
if not non_stale:
    print(f"BLOCKED: {spec['id']} has 0 linked non-stale tests. GOV-12 requires at least 1 test.")
    print("Create tests with /kb-work-item before promoting.")
    # DO NOT PROMOTE — exit here
```

## Step 4: Execute Promotion

If all validations pass:
```python
updated = db.update_spec(
    "$0", changed_by="Claude",
    change_reason="SXXX: Promoted to <status> -- all assertions pass, N tests pass",
    status="<target_status>",
)
```

If validations fail, do NOT promote. Create work items for each blocker using `/kb-work-item`.

## Step 5: Record Assertion Run

```python
db.insert_assertion_run(
    spec_id="$0", spec_version=spec['version'],
    overall_passed=all_passed,
    results=[{"assertion": a["description"], "passed": result, "detail": "..."}
             for a, result in zip(assertions, results)],
    triggered_by="kb-promote skill",
)
```

## Batch Promotion

List candidates via `db.list_specs(status="specified")` or `db.list_specs(status="implemented")`. Run validation for each. Report summary table of promoted vs blocked.
