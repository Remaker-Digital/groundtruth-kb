# CODEX Insight Dropbox

Purpose: store session-level Loyal Opposition outputs for Prime Builder consumption.

## Naming

- Insight files: `INSIGHTS-YYYY-MM-DD-HH-mm.md`
- The timestamp should reflect local project time at report creation.

## Minimum Content

- scope reviewed
- findings by severity (P0-P3)
- supporting evidence references
- recommended actions
- open decisions requiring owner or Prime Builder input

## Ground Rules

- Additive-only history; do not rewrite prior reports.
- Link to code/config/doc evidence directly when possible.
- Keep sensitive secrets redacted.
