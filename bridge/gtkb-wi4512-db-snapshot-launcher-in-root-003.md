# WI-4512 Implementation Report: GTKB-DbSnapshot launcher moved out of %TEMP% to an in-root home

## Summary

Implemented the GO'd proposal (`-001`, GO at `-002`) within the declared `target_paths`. The scheduled-task launcher generated by `scripts/install_db_snapshot_task.ps1` no longer lives under `$env:TEMP`; it is written to a durable in-root location, `(<ProjectRoot>)\.gtkb-state\db-snapshot\gtkb_db_snapshot_task.py`. The scheduled task's `-Argument` and the final `Script:` status line both reference the new in-root launcher. The launcher's parent directory is created before write. The snapshot OUTPUT path (`%LOCALAPPDATA%\gtkb-snapshots`, governed by the separate DB-Snapshot Output Exception) is unchanged, as are the schedule, the database, and retention behavior.

## Changes Made

### `scripts/install_db_snapshot_task.ps1` (source; `fix:`)

- Removed `$tempScript = Join-Path $env:TEMP "gtkb_db_snapshot_task.py"`.
- Added an in-root launcher: `$launcherDir = Join-Path $ProjectRoot ".gtkb-state\db-snapshot"`; `$launcherScript = Join-Path $launcherDir "gtkb_db_snapshot_task.py"`.
- Added `New-Item -ItemType Directory -Force -Path $launcherDir | Out-Null` **before** `Set-Content`, so the parent directory always exists at write time.
- `Set-Content` now targets `$launcherScript`.
- The scheduled task `-Argument` now references `"`"$launcherScript`""`.
- The final `Write-Host "Script: ..."` now prints `$launcherScript`.
- Added a `# WI-4512:` explanatory comment block distinguishing the launcher (in-root active dependency) from the snapshot output exception.

Diff stat: `1 file changed, 10 insertions(+), 4 deletions(-)`.

### `platform_tests/scripts/test_db_snapshot_launcher_in_root.py` (new test; reported under the `fix:` change set)

New static-assertion regression test reading the installer source. It strips PowerShell comments first (`_installer_code()`), so the assertions target executable code rather than prose that may legitimately mention `$env:TEMP`.
