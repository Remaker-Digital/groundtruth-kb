## 0. What This Proposal Is

Slice 1 of GTKB-WRAPUP-ENHANCEMENTS. Three independent sub-items, all
non-mutating, all driven by the on-demand `/wrap` trigger (per
DECISION-0006: "On-demand /wrap only (Recommended)"):

1. **W0 — Transcript-snapshot precursor.** A small capture script that
   freezes the active session transcript to a deterministic location
   under `.groundtruth/session/snapshots/<session-id>/` so downstream
   scanners can read it without harness-specific path derivation.
2. **W1 — S5 hygiene scanner.** Detects drift classes that bit S304
   (uncommitted layered drift), S307 (worktree drift / hardcoded
   paths), and S308–S309 (untracked bridge files, stale `.pyc`
   parallel-checkout artifacts, INDEX-vs-disk mismatches at the file-
   existence level).
3. **W2 — S2 cross-artifact consistency scanner.** Detects the
   phantom-INDEX-citation class that bit S308 multiple times: INDEX.md
   asserts `VERIFIED: bridge/x-006.md` when only `-001.md` exists on
   disk; work_list.md cites bridge files that aren't there;
   MEMORY.md indexes topic files that aren't there.

Net wrap-up impact: two non-mutating scanners with structured findings
output, runnable as `python scripts/wrap_scan_hygiene.py` and
`python scripts/wrap_scan_consistency.py`, plus a precursor capture
script. Owner reviews findings; `kb-session-wrap` proceeds with mutating
phases only after the scanners report clean (or the owner explicitly
acknowledges open findings).

Net reliability impact: closes two recurring incident classes
(audit-trail gap, phantom-INDEX citation) at session end where context
is freshest, rather than discovering them next-session at session start.

## 2. Implementation Scope

### 2.1 W0 — Transcript-snapshot precursor

**New file:** `scripts/wrap_capture_transcript.py`.

```python
"""Freeze the active session transcript to a deterministic snapshot
location so scanners (W1 hygiene, W2 consistency, future W3
continuation, W4 synthesis) can read it without harness-specific path
derivation.

Per DECISION-0006, wrap scanners run on owner-triggered `/wrap`, not on
Stop. The transcript path is harness-specific (Claude Code uses
`~/.claude/projects/<encoded-project>/<session-id>.jsonl` typically).
Slice 1 captures the snapshot and writes a small manifest pointing to
it; downstream scanners read manifest first, then transcript if needed.

Output (idempotent, atomic via write-to-.tmp + os.replace):
- .groundtruth/session/snapshots/<session-id>/manifest.json
  {
    "session_id": "S310",
    "captured_at": "2026-04-25T23:55:12Z",
    "transcript_source": "<absolute path or null if not derivable>",
    "transcript_snapshot": "<relative path under snapshots/ or null>",
    "git_head": "<sha>",
    "uncommitted_paths": [...],
    "branch": "develop"
  }
- .groundtruth/session/snapshots/<session-id>/transcript.jsonl
  (only when transcript_source is locatable; copy, not symlink)

Skips gracefully when transcript path cannot be derived. The manifest
is always written so scanners have a deterministic input file even
when the transcript is unavailable.
"""
```

**Behavior:**
- Accepts `--session-id <id>` (required) and optional `--snapshot-root
  <path>` (defaults to `.groundtruth/session/snapshots`).
- Derives the harness transcript path via the same encoding rule used
  by `tests/scripts/test_memory_md_ceiling.py::_user_auto_memory_path`
  (drive + path-separator-replaced project root). Falls back to env
  var `CLAUDE_TRANSCRIPT_PATH` if set.
- Captures git HEAD sha and `git status --porcelain` output into
  manifest. These are the inputs W1 and W2 actually consume; the
  transcript jsonl is auxiliary for future S1/S4 use.
- Atomic writes via `_atomic_write_text()` (helper extracted from the
  S309 P1 helper into a shared `scripts/_wrap_io.py` module — see
  §2.4).
- Exits 0 on success; non-zero only on filesystem errors. Missing
  transcript is logged to stderr but is not a failure.

**Out of scope for Slice 1:** transcript redaction, transcript
diffing, snapshot retention/cleanup policy, multi-harness reconciliation
(Codex transcript path encoding will follow in a Codex-parity slice).

### 2.2 W1 — S5 hygiene scanner

**New file:** `scripts/wrap_scan_hygiene.py`.

Detects drift classes that have recurred:

| Check | Detects | Past incident |
|---|---|---|
| `git_uncommitted_paths` | Modified-but-not-staged + staged-but-not-committed | S304 layered drift |
| `git_untracked_in_tracked_dirs` | New files in `bridge/`, `memory/`, `scripts/`, `.claude/` not in `.gitignore` | S294 blanket-ignore class |
| `bridge_files_not_in_index` | Files in `bridge/` not referenced by any `bridge/INDEX.md` line | S308 dangling -016/-017 (caught by manual reconciliation; this catches it mechanically) |
| `pyc_without_source` | `__pycache__/*.pyc` whose corresponding `.py` source is missing | S308 stale parallel-checkout `work_subject_audit.cpython-XX.pyc` |
| `tmp_artifacts_in_repo` | `*.tmp`, `.tmp.driveupload/`, `*.bak`, `*.orig` files ≥ 24h old | S305 387 GB `.tmp.driveupload` cleanup precedent |
| `hardcoded_old_project_root` | Files referencing the old `E:\Claude-Playground\CLAUDE-PROJECTS\Agent Red Customer Engagement\` root | S307 `feedback_no_hardcoded_paths.md` |

**Output:**
- Emits structured JSON to stdout (machine-readable for skill consumption).
- Optional `--report-format markdown` for human review.
- Optional `--write-report <path>` writes to file (atomic).
- Findings have severity: `info` (notable but expected), `warn` (should
  resolve before next session), `error` (must resolve before
  `kb-session-wrap` Phase 1).
- Exit code: 0 if all findings ≤ `info`; 1 if any `warn`; 2 if any
  `error`. Skill consumes exit code to gate Phase 1.

**Non-mutating:** the scanner only reads. It does not stage, delete,
move, or commit anything. All remediation is up to Prime + Codex
review.

**Out of scope for Slice 1:** auto-remediation; cross-repo drift
(parallel `groundtruth-kb` checkout state); .gitignore-aware filtering
beyond the standard `.gitignore` parse (sub-repo `.gitignore` chains
not honored — flagged as known limitation in the script header).

### 2.3 W2 — S2 cross-artifact consistency scanner

**New file:** `scripts/wrap_scan_consistency.py`.

Detects the phantom-INDEX class and adjacent reference-integrity
failures:

| Check | Detects | Past incident |
|---|---|---|
| `index_cites_missing_bridge_file` | `bridge/INDEX.md` lines (NEW/REVISED/GO/NO-GO/VERIFIED) pointing at files that don't exist on disk | S308 `gtkb-dashboard-industry-alignment-slice2` (VERIFIED at -006, only -001 existed) |
| `bridge_file_orphaned_from_index` | Files in `bridge/` whose path is not on any line in `bridge/INDEX.md` | (overlap with W1 `bridge_files_not_in_index`; W2 reports it as a consistency-side defect with the orphan's `Document:` group context) |
| `worklist_cites_missing_bridge_file` | `memory/work_list.md` references to `bridge/<name>-<NNN>.md` that don't exist | (latent class; not yet incident-recorded but high-risk after S308 reconciliation churn) |
| `memory_md_cites_missing_topic_file` | User-auto-memory `MEMORY.md` index entries `[Title](file.md)` where `file.md` doesn't exist under `memory/` | (latent class; closes after S309 P1 trim) |

**Output:** same structured-JSON + markdown + severity + exit-code

**Non-mutating:** read-only over `bridge/`, `memory/`, `.claude/`,
`groundtruth.db`. No writes to KB.

**Out of scope for Slice 1:** reverse-direction checks (e.g., DA records
that *should* exist for a bridge thread but don't — that's S3
loose-ends territory in a later slice); semantic consistency (e.g.,
INDEX claims VERIFIED but file content says NEW — a content scan, not
a reference-integrity scan).

### 2.5 `/wrap` skill scaffold

**New file:** `.claude/skills/kb-session-wrap-scan/SKILL.md`.

Slice 1 does **not** modify the existing
`.claude/skills/kb-session-wrap/SKILL.md`. It introduces a new
narrowly-scoped sibling skill `kb-session-wrap-scan` whose only job is:

1. Run `scripts/wrap_capture_transcript.py --session-id <derived>`.
2. Run `scripts/wrap_scan_hygiene.py --report-format markdown
   --write-report .groundtruth/session/snapshots/<id>/wrap-scan-hygiene.md`.
3. Run `scripts/wrap_scan_consistency.py --report-format markdown
   --write-report .groundtruth/session/snapshots/<id>/wrap-scan-consistency.md`.
4. Print both report bodies inline.
5. Exit. **Does not run any mutating wrap-up step.** Owner reviews
   findings, then explicitly invokes `/kb-session-wrap` for the
   mutating phases.

Trigger: standalone `/wrap-scan` slash invocation. The user-facing
contract is "scan, don't mutate." Slice 2 (W3/W4 continuation +
synthesis) wires this into a unified `/wrap` skill that runs scan first
and prompts the owner before mutation. Slice 1 keeps the two skills
separate so the scan path can ship and be exercised before the unified
flow lands.

**Why a separate skill rather than extending kb-session-wrap?** Per
DECISION-0007 ("Separate item, coordinated phases"), this is the
coordinated-phase boundary. The mutating skill stays untouched; the
scan skill is additive; Slice 2 unifies. This keeps Slice 1 reversible
(remove one new skill + three new scripts; existing wrap-up unaffected).

## 5. Risk Analysis

### 5.1 Failure modes for the change itself

- **Helper extraction breaks `session_self_initialization.py` callers.**
  Mitigated by step 2 in §4: the four call sites are exercised by
  `tests/scripts/test_session_self_initialization.py`, which runs in
  the same regression sweep. The relocation is a pure import-path
  change; behavior is identical.
- **Transcript path derivation fails on Codex / non-Windows.**
  Mitigated by §2.1 graceful skip + the `CLAUDE_TRANSCRIPT_PATH` env
  var override. Codex transcript discovery is explicitly out of scope
  for Slice 1.
- **W1 / W2 produce false positives on the live repo.**
  Mitigated by exit-code severity gating: only `error`-level findings
  block `kb-session-wrap`; `warn` and `info` are advisory. Owner can
  always proceed by ignoring scanner output (the scan skill is
  separate from the mutating skill — no automation forces it).
- **Synthetic-fixture tests pass while live-repo behavior diverges.**
  Mitigated by step 10 in §4: manual smoke run on the live repo with
  output review before committing.
- **Phantom-INDEX detection mis-classifies the existing S308 INDEX
  comment-block reconciliations** (the deliberately-removed phantom
  lines). Mitigated by W2 reading INDEX.md as it currently is (post-
  reconciliation); the comment blocks at lines 97-110, 173-182, and
  237-252 are documentation, not active citations. The scanner parses
  status-line patterns (`STATUS: bridge/...md`), not free-form
  comments.

### 5.2 Failure modes the change prevents

- Audit-trail-gap class (S308–S309 recurring): bridge files referenced
  in INDEX but missing from disk; bridge files on disk but missing
  from INDEX. W1 and W2 catch both directions at session end.
- Phantom-INDEX-citation class (S308 incidents in
  `gtkb-dashboard-industry-alignment-slice2`,
  `gtkb-membase-effective-use-umbrella`,
  `gtkb-dashboard-industry-alignment-slice2b-metrics`): INDEX claims
  VERIFIED at a version that never landed. W2 catches mechanically.
- Layered uncommitted drift (S304 incident at `c6882c9d`): three
  layers of uncommitted state silently accumulating across sessions.
  W1 surfaces at session end; owner decides whether to commit or
  revert before next session.
- Worktree drift / hardcoded paths (S307 incidents): old project root
  references in scripts and config. W1 `hardcoded_old_project_root`
  catches mechanically.

### 5.3 Rollback

- W0 / W1 / W2 scripts: delete the three files; remove imports from
  `_wrap_io` (collapse helper back into `session_self_initialization.py`
  if desired). No persisted state to clean up — snapshot directories
  under `.groundtruth/session/snapshots/` are gitignored and disposable.
- Scan skill: delete `.claude/skills/kb-session-wrap-scan/SKILL.md`.
- Tests: delete the three test files; remove from
  `release_candidate_gate.py` pytest list.
- Helper relocation: revert to inline definition in
  `session_self_initialization.py`; delete `scripts/_wrap_io.py`.

All rollback steps are mechanical and reversible. No KB schema or
external system changes.
