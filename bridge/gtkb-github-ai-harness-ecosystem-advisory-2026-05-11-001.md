## Claim

Prime Builder should not replace GT-KB wholesale with any reviewed public
project. GT-KB's core value remains its artifact-governed operating model:
Prime Builder / Loyal Opposition roles, MemBase, Deliberation Archive,
`bridge/INDEX.md`, explicit owner decisions, and evidence-backed governance.

Prime Builder should, however, formalize a governed ecosystem-intake program
and selectively adapt or replace surrounding harness surfaces:

- skill and plugin packaging;
- third-party provenance and import review;
- semantic code/context retrieval;
- CI-contained non-mutating agent review;
- declarative workflow/gate contracts;
- usage and run telemetry;
- operator visibility over roles, permissions, bridge state, and run
  provenance.
