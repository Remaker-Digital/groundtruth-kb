## Known Issue — BridgeBackgroundLauncher

`repair-permanent-bridge-automation.ps1` (TRACKED, modified in this
worktree) adds to its `$RequiredFiles` list:

- `BridgeBackgroundLauncher.cs` (C# source, ~6.5 KB, exists untracked)
- `BridgeBackgroundLauncher.exe` (compiled binary, ~8.7 KB, exists untracked)

Both files are currently SILENTLY IGNORED by `.gitignore:213`
(`bridge-automation/*` blanket). A fresh clone has neither file; yet the
repair script throws `"Missing bridge automation file: $path"` if they're
absent.

This means committing the tracked-modified `repair-permanent-bridge-automation.ps1`
would introduce a broken `$RequiredFiles` check. Resolving it properly
requires either:

- **Option A:** Track `.cs` (source), leave `.exe` ignored, and teach the
  repair script to compile on first run via `Add-Type -Path ... -OutputAssembly ...`.
- **Option B:** Remove both entries from `$RequiredFiles` and let the
  repair script fall back to spawning PowerShell directly (no C# launcher).
- **Option C:** Track both `.cs` and `.exe` (explicitly approve binary in git).

Phase 2 does NOT commit `repair-permanent-bridge-automation.ps1` to avoid
this landmine. Phase 2b (see below) will pick one of these options via
Codex review.

## Safeguards

1. **No source code outside bridge-automation/**: `git diff --stat -- src/ tests/ scripts/` empty post-stage.
2. **No deletions.** Pure add + modify. No `git rm`.
3. **Pre-commit guardrails pass.** Run assertion ratchet (no test changes), credential scan,
   architectural guards, TSX gate, test deletion guard — all no-op for this scope.
4. **PS1 syntax validation.** Run the `.githooks/pre-commit-ps1-parse.ps1` locally on the
   7 untracked source files before commit to confirm no parse errors. Evidence will be in
   the post-impl report.
5. **`.gitignore` pattern verification.** After `.gitignore` edit, run
   `git check-ignore -v` on both `*.generated.ps1` paths to confirm they're still ignored,
   and on one committed `.ps1` path to confirm it stays tracked.
