## Skill Design

### Claude-side SKILL.md structure

Frontmatter (precedent: `.claude/skills/bridge/SKILL.md`):

```yaml
---
name: gtkb-hygiene-sweep
description: Orchestrate the deterministic `gt hygiene sweep` CLI; classify findings by class AND by artifact lifecycle trigger category per DCL-ARTIFACT-LIFECYCLE-TRIGGERS-001; guide owner-gated remediation child-bridge filing. Use when investigating config-drift class observations, after seeing repeated similar config-defect bridges in a session, or proactively when the session-start hook surfaces a class-observation threshold. The companion CLI `gt hygiene sweep` (WI-3420 VERIFIED) provides the deterministic discovery; this skill provides the operator-facing classification + owner-AUQ workflow.
---
```

Body sections (per scoping bridge Component 1 + `DCL-ARTIFACT-LIFECYCLE-TRIGGERS-001` derived discipline):

1. **When to invoke** - explicit owner direction; after 2+ similar config-defect bridges in a session; S363-class class-observation thresholds.
2. **What this skill does** - invokes `gt hygiene sweep`; reads `findings.json`; classifies findings by class AND by artifact lifecycle trigger category before presenting owner-AUQ options.
3. **Mandatory pre-flight** - read live INDEX; never auto-file remediation bridges without owner AUQ approval; never silently advance artifact lifecycle states.
4. **Workflow** - invoke CLI → read JSON → **classify each finding by lifecycle trigger category (deferred / blocked / superseded / unresolved-new / verified-stable) per `DCL-ARTIFACT-LIFECYCLE-TRIGGERS-001` before recommending action** → present AskUserQuestion menu of remediation options with explicit-state-preserving framing → file child-bridges only on owner AUQ approval. **Per `DCL-ARTIFACT-LIFECYCLE-TRIGGERS-001`, the skill never silently transitions artifact lifecycle states; every state transition flows through an explicit owner-decision capture.**
5. **Output** - findings count by class AND by lifecycle trigger category; ranked remediation options; per-finding child-bridge slug suggestions with explicit-state labels.
6. **Does NOT** - auto-file bridges; expand pattern set without owner approval; modify source files directly; **silently transition artifact lifecycle states without owner AUQ approval (per `DCL-ARTIFACT-LIFECYCLE-TRIGGERS-001`)**.
