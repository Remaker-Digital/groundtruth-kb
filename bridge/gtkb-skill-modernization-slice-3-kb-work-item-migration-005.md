## Implementation Plan

### New verb: `gt backlog add-work-item` (GOV-12/13 chain, fail-closed) — unchanged from -003

`cli_backlog_add_work_item.py` composes existing primitives; non-dry-run order is fail-closed (no orphan test):

1. Validate work-item fields (delegate to `add_backlog_item`) + test fields (`--test-title`, `--test-type` ∈ {assertion,e2e,integration,unit,manual}, `--test-expected-outcome`, `--test-spec-id` defaulting to `--source-spec-id`) + REQUIRED `--test-plan-phase`. Missing phase → non-zero exit, no mutation.
2. Validate the phase resolves to a current `test_plan_phases` row → non-zero exit, no mutation if not.
3. Resolve attribution fail-closed → non-zero exit before any write if unresolved.
4. Create work item (reuse `add_backlog_item`).
5. Create linked test (`db.insert_test`, monotonic `TEST-NNNN`).
6. Append the test id to the phase's `test_ids` (new append-only phase version).

`--dry-run` does steps 1-2 + reports allocations, no writes. Output JSON: `work_item_id`, `test_id`, `phase_id`. Registered under the `backlog` group.

### Canonical skill rewrite + full adapter regeneration (parity-preserving)

1. Edit canonical `.claude/skills/kb-work-item/SKILL.md`: remove inline `db.insert_work_item`/`db.insert_test` snippets (Steps 3-4); replace with `gt backlog add-work-item ... --test-plan-phase ...`. Retain GOV-12/13 narrative + taxonomy ref (skill supplies the PLAN-001 phase).
2. Regenerate the Codex adapter: `python scripts/generate_codex_skill_adapters.py --update-registry` → rewrites `.codex/skills/kb-work-item/SKILL.md` + `.codex/skills/MANIFEST.json` + refreshes the Codex kb-work-item registry source-hash. Idempotent generator: only kb-work-item changes.
3. Regenerate the Antigravity adapter: `python scripts/generate_antigravity_skill_adapters.py --update-registry` → rewrites `.agent/skills/kb-work-item/SKILL.md` + `.agent/skills/MANIFEST.json` + refreshes the Antigravity kb-work-item registry source-hash.
4. Adapters are never hand-edited; only the canonical source is edited, then generators run.

### Parity verification (acceptance gate)

Post-implementation report must show PASS for all of:
- `python scripts/generate_codex_skill_adapters.py --check --update-registry` (exit 0, adapters current).
- `python scripts/generate_antigravity_skill_adapters.py --check --update-registry` (exit 0).
- `python scripts/check_harness_parity.py --all --markdown` (Overall status: PASS; no kb-work-item regression from the baseline PASS:70).
