## Proposal

### Change 1 — Add mypy CI step to `ci.yml` `test-base` job

*(Unchanged from original — Codex approved this subpart.)*

**Touchpoint:** `.github/workflows/ci.yml`, between `Lint with ruff` (line 45) and `Run full test suite (base state)` (line 50).

**Diff sketch:**

```yaml
      - name: Lint with ruff
        run: |
          ruff check .
          ruff format --check .

      - name: mypy --strict on public API surface
        run: |
          python -m mypy --strict \
            src/groundtruth_kb/db.py \
            src/groundtruth_kb/config.py \
            src/groundtruth_kb/cli.py \
            src/groundtruth_kb/gates.py

      - name: Run full test suite (base state)
        run: pytest -v --tb=short
```

**Rationale:**
- Runs on ALL 3 Python matrix versions (3.11, 3.12, 3.13). mypy 1.20.1 is pinned in `[dev]` extra.
- Runs BEFORE pytest so type regressions fast-fail before the 638-test suite.
- Does NOT add mypy to `test-search` or `test-cross-platform`.
- File list matches `tests/test_public_api_type_checks.py:21` exactly.

### Change 2 — Ratchet `docstring-coverage.yml`: bump versions + threshold 50 → 64

**Touchpoint:** `.github/workflows/docstring-coverage.yml`

**Diff sketch:**

```diff
-      - uses: actions/checkout@v4
+      - uses: actions/checkout@v6
       ...
-      - name: Set up Python
-        uses: actions/setup-python@v5
+      - name: Set up Python
+        uses: actions/setup-python@v6
       ...
-          interrogate src/groundtruth_kb/ --fail-under 50 -vv
+          interrogate src/groundtruth_kb/ --fail-under 64 -vv
```

**Rationale:**
- Closes the Action version drift left from `gtkb-phase4b-housekeeping` (missed this workflow).
- Ratchets interrogate gate from 50 → 64, locking in Phase 4B.3's docstring additions. 1.3pp below Codex-measured 65.3% to absorb minor platform variance.

### Change 3 — Add per-file coverage gates to `ci.yml` `test-base` job using built-in coverage commands

**Touchpoint:** `.github/workflows/ci.yml` (same file as Change 1).

**Replaces the custom script** with four built-in `coverage report` calls, using the `.coverage`
data file already created by pytest-cov. Codex provided this exact pattern in the NO-GO review.

**Diff sketch:**

```yaml
      - name: Run full test suite (base state)
        run: |
          pytest -v --tb=short \
            --cov=groundtruth_kb \
            --cov-branch \
            --cov-report=term-missing
          python -m coverage report --include="*/src/groundtruth_kb/db.py" --fail-under=70
          python -m coverage report --include="*/src/groundtruth_kb/cli.py" --fail-under=72
          python -m coverage report --include="*/src/groundtruth_kb/config.py" --fail-under=92
          python -m coverage report --include="*/src/groundtruth_kb/gates.py" --fail-under=92
```

**Rationale:**
- No new script. No new test. Coverage.py `coverage report --include` is the idiomatic per-file gate.
- Thresholds anchored to Codex-measured current values with 1–3pp headroom.
- No `--cov-fail-under=0` (removed as Codex instructed — it was a no-op).
- `.coverage` file is already present in the working directory after pytest-cov writes it.
