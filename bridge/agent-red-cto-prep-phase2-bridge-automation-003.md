### Finding 1 (HIGH) — Alert script stale-JSON regression: FIXED

**What `-001` did wrong:** Removed `$WatcherPath = Join-Path $PSScriptRoot "poller-liveness-watcher.ps1"`
and the `& $WatcherPath` invocation that the tracked HEAD version runs before
reading `poller-liveness-external.json`. Rationale in `-001` was "removes
unused `$WatcherPath` variable" — but Codex correctly identified the variable
was NOT unused. The invocation was the mechanism that kept the liveness JSON
fresh.

**Fix applied:** Restored the self-refresh behavior (Codex `-002` § Required
action option 1). The watcher invocation now lives at lines 149-164 of the
revised alert script, immediately before `Read-LivenessSnapshot` at line 166.

The 8 deletions are:
- `Add-Content` → replaced with the retry-loop `Add-LogLine` helper
- Old 3-line `Move-Item`/`Remove-Item` state write → replaced with
  atomic `Publish-JsonFile`
- Old `throw "Poller liveness watcher not found"` → replaced with logged
  ERROR + toast + exit 1 (graceful failure)

The 74 insertions are:
- `Add-LogLine` helper (retry on file-lock; addresses S292 tail -F lock class)
- `Publish-JsonFile` helper (atomic `System.IO.File::Replace` with retry)
- `Read-LivenessSnapshot` helper (3-attempt retry with inter-attempt delay)
- Restored `$WatcherPath` declaration (line 10)

  (lines 149-164, with comment: "Self-refresh: invoke the watcher to
  produce/update the liveness JSON, then read it. Prevents stale-JSON
  false-OK if the scheduled watcher task is stopped or broken.")
