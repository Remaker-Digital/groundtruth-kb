---
name: No pride of ownership in pipeline components
description: Owner explicitly rejects sunk-cost attachment to any pipeline component — adopt better alternatives without hesitation
type: feedback
---

The owner does not have pride of ownership in the pipeline implementation. If we find something better, we adopt it and discard inferior elements without hesitation. The value is in discovering the best possible pipeline, not in preserving what we built.

**Why:** The pipeline is a software factory. Attachment to specific implementations (even ones we built) introduces bias that prevents adopting superior alternatives. Who built a component is irrelevant; whether it's the best available option is all that matters.

**How to apply:** When evaluating tools, frameworks, or approaches, never use "we already built X" as a reason to reject a better alternative. Evaluate on merit: reliability, fitness for the specific problem, integration cost, and long-term maintenance burden. Apply this standard to groundtruth-kb itself — if a better specification management system emerges, evaluate it honestly.
