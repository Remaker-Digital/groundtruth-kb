Agent Red Customer Experience v2.0

Admin UI Color Palette Worksheet

Updated 2026-02-04: Colors updated per designer feedback. All "Current" values below reflect the approved changes that are now live in the prototype. Use the New Color fields for any further refinements. Items marked with changed were updated in this revision.

This worksheet shows every UI element type used in the Agent Red admin dashboard, rendered in both Dark Mode (default) and Light Mode, with the current hex color labeled on each element.

Each element includes a brief description explaining where you see it on the actual admin dashboard pages. The designer should fill in the "New Color" input field for any element she wants to change. Leave blank to keep the current value.

Instructions for the Designer:
  1. Review each element in both Dark and Light mode panels below. Each swatch includes a description of where the element appears on the admin dashboard.
  2. For any color you want to change, type your new hex value in the New Color input field next to each swatch.
  3. Leave the field blank if the current color should remain unchanged.
  4. The reference table in Section 9 also has input fields in every "New Color" cell -- you can fill in values there instead if you prefer a spreadsheet-style view.
  5. Return this file (or a PDF/screenshot with annotations) and we will apply all changes.

10. Notes & Additional Guidance

Typography: Inter (body) + JetBrains Mono (code). These are fixed and not part of this color review.

Dark mode is the default. Most merchants will see dark mode first. Prioritize dark mode polish.

Header is now always dark (#0a0a0a) in both dark and light mode. Text in the header is always light (#F5F5F5).

Warm taupe borders (#938a87): The designer introduced a warm taupe border color at 0.5px width. This replaces the previous rgba(255,255,255,0.08) semi-transparent borders. If you want to adjust the warmth or opacity, note the direction (warmer/cooler) so we can apply consistently.

Accessibility: All text colors must maintain WCAG AA contrast (4.5:1 for body text, 3:1 for large text) against their background.

Brand primary (#C41E2A): This is the approved brand color. Changes here affect the entire product identity -- logo, buttons, accents, charts, widget header.

Nav/selection states now neutral: Active nav background, selected conversation background, and selected item borders now use neutral white-at-opacity instead of red tints. If you want to reintroduce color, specify the exact rgba value.

Returning the worksheet: Fill in only the values you want changed. You can return this HTML file with values typed in, a PDF with annotations, or a simple spreadsheet mapping "Token Name -> New Color." Any format works.
